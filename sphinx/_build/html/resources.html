User¶

Resource Operation Description
Usuario POST /user Login/Registrar un nuevo usuario
POST /user¶

Registers a new user manually given its body parameters

Example request:

POST /user HTTP/1.1
Host: gokartmania.com.mx
Accept: application/json
Content-Type: application/json

{
    "name": "Josep Romagosa",
    "email": "josepromll@gmail.com"
}

Example response:

HTTP/1.1 200 OK
Vary: Accept
Content-Type: application/json

{
    "_id": "2d092ef7044b42708833bb4f7ab71082",
    "email": "josepromll@gmail.com",
    "name": "Josep Romagosa",
    "reservations": []
}

Example response error:

HTTP/1.1 400 Bad request
Vary: Accept
Content-Type: application/json

{
"message": "El token de acceso ha expirado, inicie sesión nuevamente",
"success": false
}
Response Headers:
 
  • Content-Type – application/json
Status Codes:
  • 200 OK – payments visualized
  • 400 Bad Request – malformed request
Return:

A brand new user, or a session to the currently existing user
